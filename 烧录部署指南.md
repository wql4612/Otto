# 桌面机器人 — 烧录部署指南

> **版本**: v3 (阶段 A+B 整合版)  
> **日期**: 2026-06-19  
> **硬件**: Seeed XIAO ESP32S3 Sense + STC89C52 (51 MCU)

---

## 一、项目概述

基于 ESP32S3 的桌面机器人，全本地推理。功能包括:

- **智能灯控**: 检测到人→自动开灯+欢迎，人走→延时关灯
- **挥手控制**: 对摄像头挥手→切换台灯状态
- **Web 控制面板**: 浏览器实时预览摄像头、控制设备、查看日志
- **LCD 表情**: 7 种动态表情 (开心/待机/聆听/惊讶/睡觉/困惑/卖萌)
- **舵机动画**: 待机微动 + 欢迎点头 + 互动招手
- **语音识别**: 框架就绪，Edge Impulse 模型训练完成后激活

## 四、Arduino IDE 环境准备

### 4.1 安装开发板支持

1. 打开 Arduino IDE → 文件 → 首选项
2. 附加开发板管理器网址: `https://espressif.github.io/arduino-esp32/package_esp32_index.json`
3. 工具 → 开发板 → 开发板管理器 → 搜索 `esp32` → 安装 **esp32 by Espressif Systems** (推荐 v3.0.x)

### 4.2 安装依赖库

工具 → 管理库，搜索并安装:

| 库名 | 作者 | 用途 |
|------|------|------|
| **Adafruit GFX Library** | Adafruit | 图形基础库 |
| **Adafruit ST7789** | Adafruit | LCD 驱动 |
| **JPEGDEC** | Larry Bank | JPEG 解码 (表情显示) |

### 4.3 安装 LittleFS 上传插件

1. 下载 [ESP32 LittleFS Upload 插件](https://github.com/lorol/arduino-esp32fs-plugin/releases)
2. 将 `ESP32FS.jar` 放入 Arduino IDE 安装目录下的 `tools/ESP32FS/tool/` 文件夹
3. 重启 Arduino IDE，工具菜单中出现 `ESP32 LittleFS Upload`

## 五、烧录步骤

### 步骤 1: 修改 WiFi 配置

打开 `AI_Desktop_Assistant/main/main.ino`，修改第 36-37 行:

```cpp
const char* WIFI_SSID     = "你的WiFi名称";
const char* WIFI_PASSWORD = "你的WiFi密码";
```

### 步骤 2: 选择开发板配置

Arduino IDE → 工具:

| 设置项 | 值 |
|--------|-----|
| 开发板 | **Seeed XIAO ESP32S3 Sense** |
| PSRAM | **OPI PSRAM** |
| Partition Scheme | **No OTA (Large APP)** 或 **Huge APP (3MB No OTA/1MB SPIFFS)** |
| USB CDC On Boot | **Enabled** |
| CPU Frequency | 240MHz |

### 步骤 3: 上传 LittleFS 文件

1. 确认 `AI_Desktop_Assistant/main/data/` 目录存在，包含:
   - `control_panel.html`
   - `faces/face_happy.jpg`
   - `faces/face_idle.jpg`
   - `faces/face_listening.jpg`
   - `faces/face_surprised.jpg`
   - `faces/face_sleep.jpg`
   - `faces/face_confused.jpg`
   - `faces/face_cute.jpg`

2. 用 USB-C 线连接 XIAO ESP32S3 Sense 到电脑

3. 工具 → **ESP32 LittleFS Upload** → 等待完成

   > 如果上传失败，检查:
   > - 串口是否选对
   > - PSRAM 是否开启
   > - 分区表是否有足够的 SPIFFS/LittleFS 空间

### 步骤 4: 编译上传固件

1. 打开 `AI_Desktop_Assistant/main/main.ino`
2. 点击 **上传** (→ 箭头)
3. 等待编译 + 烧录完成 (约 1-3 分钟)
4. 打开串口监视器 (115200 baud)，应看到:

```

[Vision] Ready (JPEG sampling + frame diff)
[Voice] Mic OK (PDM 16kHz)
[Voice] Bypass mode — no EI model loaded
[LittleFS] Total: 1472 KB, Used: 58 KB
[WiFi] Connecting....
[WiFi] Connected! IP: 192.168.x.x
[Server] Started on port 80

  >>> http://192.168.x.x <<<
```

### 步骤 5: 验证

1. 浏览器打开串口打印的 IP 地址
2. 应看到深色主题的控制面板，包含:
   - 实时摄像头画面
   - 人体检测状态指示
   - 台灯/风扇/音响设备状态
   - 手动控制按钮
   - 事件日志
3. 在摄像头前走动 → 状态面板应显示"检测到人"→ LCD 显示 😊
4. 挥手 → LCD 显示 😲 → 台灯状态切换

## 六、51 MCU 烧录 (可选)

51 MCU 负责实际的继电器控制和 433MHz RF 编码。如果暂时没有 51 MCU，ESP32 可独立运行（灯控等状态仅在软件层面更新）。

### 烧录 51 MCU 固件

1. 安装 Keil C51 或使用 STC-ISP 烧录工具
2. 打开 `AI_Desktop_Assistant/mc51_firmware.c`
3. 编译生成 .hex 文件
4. 用 USB-TTL 下载器连接 51 MCU:
   - TXD → P3.0 (RX)
   - RXD → P3.1 (TX)
   - VCC → 5V
   - GND → GND
5. 冷启动下载 (STC-ISP → 点击下载 → 给 51 MCU 上电)
6. 下载完成后，LED 闪烁 3 次表示启动成功

### 学习 433MHz 遥控码

1. ESP32 上电，确保 UART 连接正常
2. 浏览器访问 `http://<ESP32_IP>/api/action?cmd=rf_on`
3. ESP32 通过 UART 发送 `RF_SEND(slot=0)` 命令给 51 MCU
4. 51 MCU 通过 433MHz TX 发射已学习的编码
5. 如需学习新遥控器: 将遥控器对准 433M RX 模块按键，同时 ESP32 发送学习命令

## 八、故障排除

| 症状 | 检查项 |
|------|--------|
| 编译报错 `JPEGDEC.h not found` | 库管理器安装 **JPEGDEC** |
| 编译报错 `Adafruit_ST7789.h not found` | 库管理器安装 **Adafruit ST7789** 和 **Adafruit GFX** |
| 编译报错 PSRAM 相关 | 工具 → PSRAM → **OPI PSRAM** |
| 上传后串口无输出 | 工具 → USB CDC On Boot → **Enabled** |
| 摄像头无画面 | PSRAM 已开启？摄像头连接松动？ |
| WiFi 连不上 | SSID/密码正确？2.4GHz WiFi？ |
| 网页打不开 | ESP32 和电脑在同一网络？IP 正确？ |
| 表情不显示 | LittleFS 上传成功？JPEGDEC 库已安装？ |
| LCD 白屏/花屏 | SPI 接线正确？CS=2, DC=3, SCLK=2, MOSI=3？ |
| 舵机不动作 | 独立 5V 供电？GPIO21/44 接线正确？ |
| 舵机抽搐/重启 | 舵机堵转电流过大 → 需要独立供电 |

## 九、系统启动后的行为

1. **开机**: LCD 显示启动画面 → 舵机归中 → 扬声器播放提示音
2. **WiFi 连接**: LCD 显示 IP 地址 → 切换待机表情
3. **空闲**: 舵机每 3 秒微动 (85°~95°)，LCD 显示 😴 或 😐
4. **检测到人**: LCD → 😊，舵机点头欢迎，台灯自动亮起
5. **人离开 10 秒**: LCD → 😴，台灯熄灭，舵机归位
6. **挥手**: LCD → 😲，台灯状态切换
7. **断网**: 每 10 秒自动重连
