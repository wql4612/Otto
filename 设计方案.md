# 基于语言与视觉感知的桌面机器人与智能设备联动系统 — 设计方案 v2

## 一、项目概述

项目名：基于语言与视觉感知的桌面机器人与智能设备联动系统

**核心目标**：构建一个ESP32S3驱动的桌面机器人，全部采用**本地推理**——人体检测（SenseCraft）、语音指令识别（ESP-SR）、动作联动，通过51单片机+433MHz RF控制台灯、风扇、音响等桌面设备，实现端侧智能家居效果。

**参考项目**：[AI_DesktopCat_Qwen3.5Omni](https://github.com/AI-FanGe/AI_DesktopCat_Qwen3.5Omni)（取其硬件框架，替换为全本地方案）

**技术路线**：ESP32S3全本地推理 + 433MHz设备控制，成本~245元，1个月周期

**核心设计原则：不上云，不训练大模型，全部用现成的本地分类模型。**

### 2.2 四大数据流（全部离线）

**语音指令流：**
```
🎤麦克风持续监听 → ESP-SR唤醒词检测("小猫咪") → 后续2秒语音 → ESP-SR指令分类
→ 匹配到指令("开灯"/"关灯"/"风扇"/"音乐"等) → UART发指令给51单片机
→ 🔊喇叭播放确认音效 + 🖥LCD切换表情
耗时：<500ms，全本地
```

**人体检测流：**
```
📷摄像头 → SenseCraft Person Detection模型(本地) → 检测到人/无人
→ UART→51单片机→继电器动作→台灯开/关
→ LCD表情切换(开心/待机)
耗时：~600ms/帧，全本地
```

**视觉手势/动作流（简化版）：**
```
📷摄像头 → 帧差法检测运动区域 → 运动区域占比超过阈值
→ 触发"检测到动作"事件 → 可映射为挥手开关灯
（不做复杂手势分类，做"有运动/无运动"二分类，可靠且实用）
```

**Web控制流：**
```
手机/电脑浏览器 → WiFi → ESP32S3内嵌WebServer → JSON指令
→ 实时视频预览 + 手动设备控制 + 语音指令日志
```

### 3.2 433MHz模块说明

433MHz替代红外的优势：
- **非定向**：不需对准，全房间覆盖
- **穿透力强**：隔墙也能控制
- **编码方式相同**：学习遥控器→存储→重放，流程和红外一样
- **成本相近**：FS1000A发射模块 5元，SYN480R接收模块 4元

使用方式：
1. 用433MHz接收模块+51单片机，学习原装遥控器的RF编码
2. 存储编码到51单片机EEPROM
3. ESP32S3发指令→51单片机通过433MHz TX重放编码
4. 如果风扇/音响没有RF遥控→在电源线上串入继电器控制通断

### 4.2 ESP-SR 语音识别详解

Espressif官方的ESP-SR框架可以在ESP32S3上本地运行中文语音识别：

```cpp
// ESP-SR 集成要点
#include "esp_mn_speech_commands.h"

// 自定义指令集（存储在Flash中）
// 每个指令需要录制 20-50 个样本用于适配
esp_mn_commands_add("开灯",     CMD_LIGHT_ON);
esp_mn_commands_add("关灯",     CMD_LIGHT_OFF);
esp_mn_commands_add("开风扇",   CMD_FAN_ON);
esp_mn_commands_add("关风扇",   CMD_FAN_OFF);
esp_mn_commands_add("放音乐",   CMD_MUSIC_PLAY);
esp_mn_commands_add("停下",     CMD_STOP);
// ... 共15-20个指令

// 主循环中的语音处理
void loop() {
    // 唤醒词检测（MultiNet 模型）
    if (wake_word_detected()) {
        lcd_show_listening();   // LCD显示"聆听中"表情
        led_set(RGB_GREEN);     // 状态指示
        
        // 录制2秒音频 → 指令识别
        audio_record(2000);
        int cmd = esp_mn_recognize(audio_buffer);
        
        if (cmd >= 0) {
            execute_command(cmd);  // 执行对应动作
            play_confirm_sound();  // 播放确认音效
        }
    }
}
```

**指令集设计（15条核心指令）：**

| 类别 | 指令 | 动作 |
|------|------|------|
| 灯光 | 开灯/关灯 | 继电器控制台灯 |
| 灯光 | 亮一点/暗一点 | PWM调节台灯亮度（需MOS模块支持） |
| 风扇 | 开风扇/关风扇 | 433MHz RF控制风扇 |
| 音响 | 放音乐/停下 | 433MHz RF控制音响 |
| 互动 | 你好/过来 | 舵机点头+招手 |
| 查询 | 现在几点 | LCD显示时间 |
| 模式 | 休闲模式/工作模式 | 预设场景联动 |

### 4.5 51单片机固件（Keil C51） — 负责人：组员A

**功能：**
- UART接收ESP32S3指令
- 继电器控制（台灯 + 备用通道）
- 433MHz RF编码发射（风扇/音响）
- 433MHz RF编码学习（开发阶段，接收并存储遥控器编码）

**UART通信协议（9600bps）：**
```
帧格式：STX(0xAA) | CMD(1B) | DATA_LEN(1B) | DATA(NB) | XOR_CHK(1B)

CMD指令：
  0x01 RELAY_CTRL    DATA[0]=通道(1-2) DATA[1]=0关/1开
  0x02 RF_SEND       DATA[0]=编码槽位(0-7)（预学习的遥控码）
  0x03 RF_LEARN      DATA[0]=槽位(0-7)（进入学习模式）
  0x04 STATUS_REQ    查询当前继电器状态
  
响应帧：STX(0xBB) | CMD(1B) | STATUS(1B) | XOR_CHK(1B)
```

## 六、核心技术实现要点

### 6.1 人体检测（SenseCraft本地模型）

```cpp
#include "Seeed_Arduino_SSCMA.h"

static bool person_present = false;
static unsigned long last_seen = 0;
static const unsigned long LEAVE_DELAY = 10000;  // 人走10秒后才关灯

void loop_vision() {
    if (!sscma.invoke()) return;
    
    if (sscma.boxes().size() > 0) {
        // 检测到人
        last_seen = millis();
        if (!person_present) {
            person_present = true;
            uart_send_cmd(0x01, 1, 1);           // 开灯
            lcd_show_face(FACE_HAPPY);
            servo_wake_up();                      // 唤醒动作
            play_sound(SND_GREETING);
        }
    }
    
    if (person_present && (millis() - last_seen > LEAVE_DELAY)) {
        // 确认离开
        person_present = false;
        uart_send_cmd(0x01, 1, 0);               // 关灯
        lcd_show_face(FACE_SLEEP);
        servo_rest();
    }
}
```

### 6.2 运动检测（帧差法，替代云端手势识别）

```cpp
// 简单的运动检测：对比连续两帧的差异
static uint8_t* prev_frame = nullptr;
static const float MOTION_THRESHOLD = 0.15;  // 15%像素变化=有运动

bool detect_motion(camera_fb_t* fb) {
    if (!prev_frame) {
        prev_frame = (uint8_t*)malloc(fb->len);
        memcpy(prev_frame, fb->buf, fb->len);
        return false;
    }
    
    // 下采样+灰度比较（每10个像素采样1个）
    int diff_count = 0, total = 0;
    for (int i = 0; i < fb->len; i += 10) {
        if (abs(fb->buf[i] - prev_frame[i]) > 30) diff_count++;
        total++;
    }
    
    memcpy(prev_frame, fb->buf, fb->len);
    return (float)diff_count / total > MOTION_THRESHOLD;
}
```

### 6.3 语音指令识别流程

```
上电 → ESP-SR MultiNet唤醒词模型加载(约2MB PSRAM)
     → 持续监听唤醒词
     → 检测到"小猫咪"唤醒 → LCD切换"聆听"表情
     → 录制2秒PCM音频 → 送入指令分类模型
     → 输出指令ID → 查表→执行动作 → 播放确认音效
     → 回到监听状态

全程离线，延迟<500ms
```

### 6.4 433MHz RF编码控制

```
开发阶段（学习）：
  433M接收头 → 51单片机INT0中断 → 解码原装遥控器波形
  → 存储Pulse-Width序列到EEPROM（每个按键一组编码）

运行阶段（发射）：
  ESP32 → UART → 51单片机查EEPROM → 433M TX OOK调制发射
  → 家电收到→执行动作

常用编码格式：EV1527/PT2262（学习型，直接复制波形即可）
```

## 七、风险与应对

| 风险 | 概率 | 应对方案 |
|------|------|----------|
| ESP-SR唤醒词在嘈杂环境不准 | 中 | 增加按键唤醒作为保底方案；多录样本提高准确率 |
| SenseCraft模型不兼容新固件 | 低 | 备选Edge Impulse自训练；官方文档持续更新 |
| 舵机供电不足导致重启 | 高 | 独立供电；电池选3000mAh版本 |
| 433MHz与家电不兼容 | 中 | 备选：直接用继电器控制家电通断（去掉RF环节） |
| ESP32S3 PSRAM不足(8MB) | 中 | 模型量化裁剪；ESP-SR和SenseCraft不能同时加载时做分时复用 |
| 指令录制样本数不够 | 中 | 每人录20遍×4人=80样本/指令，足够适配 |
| ESP32S3 Sense缺货 | 中 | 备选：普通ESP32S3+OV2640摄像头模块(约60元) |
