# 语音识别混合架构 — 部署与测试指南

> 基于 [语音识别混合架构方案](语音识别混合架构方案.md)，本文档面向测试人员，提供完整的部署和验证流程。

## 硬件清单

| 组件 | 说明 |
|------|------|
| XIAO ESP32S3 Sense | 主控（已烧录固件） |
| 电脑 (Windows) | 运行 Python 服务器 |
| 同一局域网 | ESP32 和电脑连同一个 WiFi |

## 一、服务器端部署

### 1.1 安装 Python 依赖

```bash
cd server
pip install -r requirements.txt
```

### 1.2 配置 API Key

在 `server/` 目录下创建 `.env` 文件：

```
DASHSCOPE_API_KEY=你的阿里云API_Key
```

### 1.3 启动服务

```bash
python app.py
```

成功输出：

```
==================================================
  桌面小猫机器人后端服务
==================================================
  ASR:        可用
  API Key:    已配置
  监听地址:   http://0.0.0.0:8765
  ASR 模型:   paraformer-realtime-v2
  对话模型:   qwen3-omni-flash
==================================================
INFO:     Uvicorn running on http://0.0.0.0:8765
```

### 1.4 验证服务

浏览器访问 `http://localhost:8765/health`，应返回：

```json
{"ok":true,"asr_available":true,"api_key_configured":true,"esp32_connected":false,"ui_clients":0}
```

## 二、ESP32 固件部署

### 2.1 编译配置

| 参数 | 值 |
|------|------|
| Board | XIAO ESP32S3 |
| PSRAM | Enabled |
| Partition Scheme | Minimal SPIFFS (或带 LittleFS 的分区) |
| USB CDC On Boot | Enabled |

### 2.2 修改配置

打开 `main.ino`，搜索 `SERVER_IP`，改成你电脑的实际局域网 IP：

```cpp
#define SERVER_IP "你的电脑IP"  // ← 必须改！用 ipconfig 查看
#define SERVER_PORT 8765         // 保持不变
```

> 同一个 WiFi 下，不同电脑的 IP 不同。每次换电脑跑服务器都需要更新这个 IP 并重新上传固件。

### 2.3 烧录

用 USB-C 连接 ESP32，点击 Arduino IDE 的 **上传**。

### 2.4 验证启动

打开串口监视器（115200 bps），依次看到：

```
=== AI Desktop Assistant v3 ===
[Voice] EI KWS model loaded (otto-wake)
[Voice] Wake word: 小猫咪
[WiFi] Connected! IP: 192.168.x.x
[Cloud] Connected to 192.168.x.x:8765
Setup complete
```

**如果 `[Cloud] Connected` 没出现**：确认电脑防火墙已放行 8765 端口，且 ESP32 和电脑在同一局域网。

## 三、功能测试

### 测试 1：唤醒词检测

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | ESP32 启动完成 | LCD 显示 IDLE 表情 |
| 2 | 说"小猫咪" | 串口打印 `[Wake] 小猫咪 (0.xx)` |
| 3 | — | LCD 切换为 LISTENING 表情 |
| 4 | — | 串口打印 `[Voice] Wake → dialog` |
| 5 | — | 服务器终端打印 `[AUDIO] START 收到` |

### 测试 2：语音指令控制

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 说"小猫咪" | 进入对话模式 |
| 2 | 说"开灯" | 服务器打印 `[CMD] 匹配指令: '开灯' → CMD_LIGHT_ON` |
| 3 | 说"关灯" | 同上，command 变为 `CMD_LIGHT_OFF` |
| 4 | 说"你好" | ESP32 执行点头动画 |

> 服务器端 `voice_command.py` 定义了 15 个中文指令关键词，匹配成功后直接返回指令给 ESP32，不走 LLM。

### 测试 3：AI 自由对话

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 说"小猫咪" | 进入对话模式 |
| 2 | 说"今天天气怎么样" | 服务器打印 `[ASR FINAL] → 今天天气怎么样` |
| 3 | — | Qwen3-Omni 生成回复文本 + TTS 音频 |
| 4 | — | ESP32 通过喇叭播放回复 |
| 5 | — | LCD 根据 AI 情绪切换表情 |

### 测试 4：对话超时

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 进入对话模式 | LCD = LISTENING |
| 2 | 15 秒不说话 | 串口打印 `[Voice] Dialog timeout` |
| 3 | — | LCD 恢复 IDLE 表情 |
| 4 | 再次说"小猫咪" | 重新进入对话模式 |

### 测试 5：断网容错

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 关闭服务器 `Ctrl+C` | — |
| 2 | 说"小猫咪" | 串口仍打印 `[Wake]`（本地 KWS 不受影响） |
| 3 | — | LCD 短暂切换 LISTENING 后恢复（云桥接连不上） |
| 4 | 重启服务器 | ESP32 自动重连（每 5 秒重试） |

## 四、通信协议参考

### ESP32 → 服务器

| 消息 | 格式 | 触发时机 |
|------|------|----------|
| 启动 ASR | 文本 `"START"` | 唤醒后 |
| 停止 ASR | 文本 `"STOP"` | 超时/手动退出 |
| PCM 音频 | 二进制，16kHz 16bit mono，640B/帧 | 对话中持续发送 |

### 服务器 → ESP32

| type | 示例 | 触发 |
|------|------|------|
| `cmd` | `{"type":"cmd","command":"CMD_LIGHT_ON"}` | 匹配到指令关键词 |
| `chat` | `{"type":"chat","text":"好的主人~"}` | AI 对话最终回复 |
| `chat_partial` | `{"type":"chat_partial","text":"..."}` | AI 增量文本（刷新超时） |
| `tts_start` | `{"type":"tts_start"}` | TTS 音频即将开始 |
| 二进制音频 | PCM 24kHz 16bit mono，960B/帧 | TTS 播放中 |
| `audio_done` | `{"type":"audio_done"}` | TTS 播放完毕 |

## 六、常见问题

### Q1: 服务器启动报 `ModuleNotFoundError: No module named 'audioop'`

你的 Python 版本 ≥ 3.13，`audioop` 已被移除。使用最新的 `server/app.py`（已兼容）。

### Q2: ESP32 编译报 `multiple definition of ...`

不要在 `main.ino` 顶部手动 `#include <otto-wake_inferencing.h>`，它会跟 `voice_driver.cpp` 冲突。已修复。

### Q3: 唤醒后不说话

- 确认服务器正在运行
- 确认 ESP32 串口显示 `[Cloud] Connected`
- 确认电脑防火墙没有阻止 8765 端口

### Q4: 唤醒准确率低

- 距离麦克风 0.5m~1m
- 发音清晰，背景噪音不要太吵
- 可以在 Edge Impulse 补充录制更多样本重新训练

### Q5: TTS 声音断断续续或无声

- 确认 ESP32 喇叭接线正确（XIAO S3 的 BCLK=6, LRCK=5, DOUT=1）
- 重新插拔喇叭模块供电
